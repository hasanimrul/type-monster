Q1:- Differences between local storage and session storage.

Answer:-

Local Storage: It is a storage or memory for store website data. User can manually store and remove the data from the storage. It is also accessible after closing the browser.
Session Storage: It is same as local storage but the only difference is session storage data got cleared when the tab or browser closed.

Q2:- Differences between global scope and block scope.

Answer:-

Global scope: Global scope contains all of the things defined outside of all code blocks. A code block simply consists of grouped statements inside curly braces ({ }). if statement, loops, function are examples of structure that create a code block. A global variable has global scope. A global variable is accessible from anywhere in the code.
Local Scope: Local scope contains things defined inside code blocks. A local variable has local scope. A local variable is only accessible where it’s declared.

Q3:- How does javascript event loop works?

Answer:-

Event Loop: The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.

Q4:- What are the reason of getting undefined in javascript?

Answer:-

Some reasons why we get undefined in JS:
=>When a variable is declared or initialized but no value is assigned to it, JavaScript automatically displays undefined.
=> When we try accessing values in an array that doesn’t exist, it will throw undefined.
=> When we try accessing values in an object that doesn’t exist, it will throw undefined.
